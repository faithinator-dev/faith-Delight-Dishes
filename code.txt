=========================================================
LOGIC EXPLANATION FOR INDEX2.HTML (MODERN FOOD UI)
=========================================================

1. INITIALIZATION & DATA FETCHING
---------------------------------------------------------
The logic begins with the `fetchFoods()` async function, which is called immediately at the bottom of the script.

- API Call: It fetches data from "https://mongotest2026.vercel.app/api/foods".
- State Management: The result is stored in a global variable `allFoods`. This allows you to filter and search the data later without making new API calls.
- UI Trigger: Once data is loaded, it calls:
  1. `displayFoods(allFoods)`: To show the grid of food cards.
  2. `loadHeroCarousel()`: To populate the sliding banner at the top.


2. IMAGE HANDLING LOGIC
---------------------------------------------------------
Since the API might not provide working image URLs, you use a local mapping strategy.

- `foodPic` Array: A hardcoded list of 29 image filenames (e.g., 'jollof.jpg').
- `getFoodImage(food)` Function:
  - Logic: It calculates an index based on the food ID (`food.id - 1`).
  - Fallback: If an image exists at that index in `foodPic`, it returns it. If not (or if the ID is higher than 29), it returns a placeholder image from "via.placeholder.com".


3. DISPLAY LOGIC (GRID SYSTEM)
---------------------------------------------------------
The `displayFoods(foods)` function is responsible for rendering the main content.

- Input: It accepts an array of food objects (either all foods or a filtered list).
- DOM Manipulation: It clears the innerHTML of the container (`#row`).
- Loop: It iterates through the array and appends a template literal string for each food item.
- Card Structure:
  - Image: Calls `getFoodImage()`.
  - Title: Displays `food.name`.
  - Buttons: 
    - "Show Details": Triggers the modal.
    - Heart Icon: Triggers the favorite logic.


4. CAROUSEL LOGIC (RANDOMIZATION)
---------------------------------------------------------
The `loadHeroCarousel()` function creates a dynamic slideshow.

- Randomization: It creates a copy of `allFoods` and sorts it randomly using `Math.random() - 0.5`.
- Selection: It slices the first 3 items from this shuffled list.
- Rendering: It generates HTML for these 3 items.
- Active State: It adds the class `active` to the first item (index === 0) so Bootstrap knows which slide to start with.


5. FILTERING LOGIC (The Complex Part)
---------------------------------------------------------
The filtering is event-driven. Event listeners are attached to the Category, Region, Vegetarian, and Spicy inputs. When any of them change, `applyFilters()` is called.

`applyFilters()` Logic:
1. Capture Values: It reads the current value of all 4 inputs simultaneously.
2. Filter Loop: It runs `allFoods.filter()` to create a new array.
3. Conditions:
   - Category: It maps your dropdown values (e.g., "main", "Soups") to the specific strings used in the API data (e.g., "Main Course", "Soup").
     * Note: It handles complex cases like "Snacks/Street Food" by checking if the API category is EITHER "Snack/Breakfast" OR "Side Dish/Snack".
   - Region: Checks for an exact string match unless "All" is selected.
   - Vegetarian: If the checkbox is checked, it only keeps foods where `food.isVegetarian` is true.
   - Spicy: If the checkbox is checked, it only keeps foods where `food.isSpicy` is true.
4. Update UI: Finally, it calls `displayFoods(filtered)` to update the screen.


6. SEARCH LOGIC
---------------------------------------------------------
- Event: Listens for the `keyup` event on the search bar.
- Real-time: As you type, it filters `allFoods` based on whether the `food.name` includes the typed text.
- Case Insensitivity: Both the food name and search text are converted to `.toLowerCase()` to ensure "Rice" matches "rice".


7. MODAL LOGIC (DETAILS POPUP)
---------------------------------------------------------
- `showDetails(id)`: Finds the specific food object by ID.
- Dynamic Content: Injects a detailed HTML structure into the modal container, including ingredients lists (mapped from the array) and preparation steps.
- Visibility: Sets the modal's CSS display property to 'flex'.
- Closing: 
  - `closeModal()` sets display to 'none'.
  - An event listener on the window detects clicks outside the modal content box to close it (backdrop click).


8. FAVORITES LOGIC (LOCAL STORAGE)
---------------------------------------------------------
This allows data to persist even if the user refreshes the page.

- Storage Key: Data is saved under the key 'favorites' in the browser's LocalStorage.
- `addFavorite(id, btn)`:
  1. Retrieves current favorites (parsed from JSON).
  2. Checks if the clicked food ID already exists in the array.
  3. Toggle Logic:
     - If exists: Removes it (splice) and changes the icon to an empty heart.
     - If new: Adds it (push) and changes the icon to a solid heart.
  4. Save: Writes the updated array back to LocalStorage.

- `showFavorites()`:
  - Reads the LocalStorage.
  - Renders a specific grid (`#favoriteRow`) with just those items.
  - Unhides the favorites section and scrolls the window down to it.


9. CSS & UI LOGIC
---------------------------------------------------------
- Dark Theme: The body background is set to a dark color (#2b2626).
- Animations: 
  - Cards hover effect (translateY).
  - Modal fade-in animation (@keyframes fadeIn).
  - Carousel zoom effect on the active image.
